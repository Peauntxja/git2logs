# 打包使用说明

## 快速开始

### macOS 用户

1. **下载并运行**
   - 下载 `GitLab提交日志生成工具.dmg`
   - 双击打开 DMG 文件
   - 将应用拖拽到"应用程序"文件夹
   - 在"应用程序"中找到并运行"GitLab提交日志生成工具"

2. **首次运行**
   - 如果系统提示"无法打开，因为来自身份不明的开发者"
   - 前往：系统设置 > 隐私与安全性
   - 点击"仍要打开"

### Windows 用户

1. **下载并运行**
   - 下载 `GitLab提交日志生成工具.exe`
   - 双击运行即可（无需安装）

2. **首次运行**
   - Windows Defender 可能会提示"未知发布者"
   - 点击"更多信息" > "仍要运行"

## 使用说明

### 基本操作

1. **填写 GitLab 信息**
   - GitLab URL: 您的 GitLab 实例地址（如：`http://gitlab.example.com`）
   - 仓库地址: 单个仓库地址（可选，如果选择"自动扫描所有项目"则不需要）
   - 访问令牌: 您的 GitLab 访问令牌（需要 `read_api` 权限）

2. **选择查询范围**
   - 分支: 指定分支名称（留空则查询所有分支）
   - 提交者: 提交者姓名或邮箱（如：`MIZUKI<mizukixja@gmail.com>`）
   - 日期范围: 选择"今天"或指定起始和结束日期

3. **选择输出格式**
   - Markdown 提交日志: 普通日志格式
   - 开发日报 (Markdown): 结构化日报格式
   - HTML 格式: 美观的 HTML 格式
   - PNG 图片: 图片格式（需要 Chrome）

4. **选择输出位置**
   - 可以输入文件路径或选择文件夹
   - 如果选择文件夹，会自动生成文件名

5. **生成日志**
   - 点击"生成日志"按钮
   - 等待处理完成
   - 查看执行日志了解进度

### 注意事项

1. **Chrome 要求**
   - 生成 PNG 图片需要系统安装 Google Chrome
   - 如果没有安装 Chrome，HTML 转图片功能将不可用
   - 其他功能（Markdown、HTML）不受影响

2. **访问令牌**
   - 需要至少 `read_api` 权限
   - 私有仓库必须提供访问令牌
   - 令牌会以 `*` 显示，点击"显示"按钮可以查看

3. **文件路径**
   - 支持相对路径和绝对路径
   - 如果输入的文件名没有扩展名，会自动添加 `.md`
   - 如果选择文件夹，会自动生成文件名

## 常见问题

### Q: 提示"未找到提交记录"
A: 请检查：
- 日期范围是否正确
- 提交者名称是否与 GitLab 中的完全匹配
- 分支名称是否正确
- 访问令牌权限是否足够

### Q: PNG 图片生成失败
A: 请确保：
- 系统已安装 Google Chrome
- Chrome 在默认安装位置
- 有足够的磁盘空间

### Q: 应用无法启动
A: 请检查：
- 系统版本是否满足要求（macOS 10.13+ 或 Windows 10+）
- 是否有足够的系统权限
- 防病毒软件是否阻止了运行

### Q: 界面显示异常
A: 请尝试：
- 重启应用
- 检查系统显示设置
- 更新系统到最新版本
